How This Telegram AI Chatbot Works

This page describes request flow, context handling, and reliability behavior for the community Grok Telegram bot.

Request flow

Telegram user message
        |
        v
Bot service (auth, quota, basic checks)
        |
        v
Grok API request
        |
        v
Bot service formats response
        |
        v
Telegram reply to user

What context is remembered

  • The bot currently keeps only the last 2 messages as short-lived context.
  • Full chat history is not stored as a long-term message database.
  • The New Chat action resets context immediately.
  • If session continuity breaks, you may need to restate key details in your next prompt.

What the bot processes

  • Telegram user identifier needed for quota, balance, and subscription checks.
  • Current request text (and short recent context) to generate a response.
  • Service metadata for billing and delivery status.

Chat UX behavior

  • Bot replies directly to the user message to keep threads readable.
  • Typing status is shown while Grok is processing the request.
  • Responses support richer formatting, including cleaner code blocks.

Available modes

  • Quick (default): low-cost everyday mode.
  • Smart, Internet, and Images modes are available with paid usage.

Latency and retries

  • Typical response latency depends on Grok API load and prompt length.
  • Transient failures may be retried automatically once.
  • If a request times out, send it again in a shorter form.
