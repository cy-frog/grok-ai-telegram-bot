Privacy and Data Handling

This page explains what data is processed to run the service and what data is not retained as long-term history.

What is stored

  • Telegram user ID (or equivalent account key) for quota and billing.
  • Balance, subscription state, and payment-related status.
  • Operational events needed for service reliability and abuse prevention.

What is not stored long-term

  • No long-term database of complete user message history.
  • No long-term database of complete model responses.
  • No persistent full transcript storage for regular usage.

Retention and short context

  • Short context from only the last 2 chat messages can be held temporarily to generate coherent replies.
  • This temporary context is not intended as permanent chat history.
  • Using New Chat resets active context for the conversation.
  • If session continuity expires, users may need to resend key context.

Processors and security

  • Requests are routed through the bot backend to Grok API infrastructure.
  • Access to operational systems is restricted to maintainers.
  • Where possible, diagnostic logs should avoid storing full message content.
