Приватность и обработка данных

На этой странице описано, какие данные обрабатываются для работы сервиса и какие данные не сохраняются как долгосрочная история.

Что хранится

  • Telegram user ID (или эквивалентный ключ аккаунта) для квоты и биллинга.
  • Баланс, статус подписки и платежные статусы.
  • Операционные события, необходимые для надежности сервиса и антиабуза.

Что не хранится долгосрочно

  • Нет долгосрочной базы полной истории пользовательских сообщений.
  • Нет долгосрочной базы полных ответов модели.
  • Нет постоянного хранения полных транскриптов при обычном использовании.

Срок хранения и короткий контекст

  • Краткий контекст из последних 2 сообщений может временно удерживаться для связных ответов.
  • Этот временный контекст не является постоянной историей чата.
  • Использование New Chat сбрасывает активный контекст диалога.
  • Если непрерывность сессии прервалась, пользователю может понадобиться повторить ключевой контекст.

Процессоры и безопасность

  • Запросы проходят через backend бота к инфраструктуре Grok API.
  • Доступ к операционным системам ограничен кругом сопровождающих.
  • Где возможно, диагностические логи не должны хранить полный текст сообщений.
